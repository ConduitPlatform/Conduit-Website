# Authorization

This module provides resource authorization for your application.<br />
It allows you to configure authorization rules that determine what actions users can perform on specific resources.<br />
The module uses a role-based access control (RBAC) system combined with resource-specific rules.

## Features

- Role-based access control (RBAC)
  - Define custom roles with specific permissions
  - Assign roles to users
  - Role hierarchy support
- Resource-level authorization rules
  - Create fine-grained access rules for resources
  - Support for CRUD operations (Create, Read, Update, Delete)
  - Rule-based evaluation engine
- Integration with Authentication module
  - Seamless user identity verification
  - Token-based authorization checks
- Administrative API
  - Manage roles and rules through the Admin Panel
  - Programmatic rule management via REST/GraphQL

## Dependencies

- [Authentication](../authentication/authentication.mdx) (required: user identity verification)
- [Database](../database/database.mdx) (required: storing roles and rules)

## Minimum Configuration

This module requires no configuration to get started.<br />
Default roles can be created through the Admin Panel or API.

## Metrics

The Authorization module exposes the following Prometheus metrics:

- `conduit_authorization_requests_total` - Tracks the total number of authorization requests
- `conduit_authorization_rules_total` - Tracks the total number of stored rules
- `conduit_authorization_roles_total` - Tracks the total number of stored roles
- `conduit_authorization_denials` - Tracks the total number of denials in authorization requests
