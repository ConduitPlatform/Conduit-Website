This guide covers deploying Conduit to a Kubernetes cluster for production use.

## Prerequisites

- Kubernetes cluster (local or cloud)
- `kubectl` installed and configured
- `helm` installed
- Docker installed (for local testing)

## Quick Start

### 1. Add Helm Repository

```bash
helm repo add conduit-platform https://conduitplatform.github.io/helm-charts
helm repo update
```

### 2. Create Values File

Create a `values.yaml` file with your configuration:

```yaml
# values.yaml
global:
  conduit:
    version: "latest"

core:
  replicas: 1
  env:
    REDIS_HOST: "redis"
    REDIS_PORT: "6379"

database:
  enabled: true
  type: mongodb
  connectionUri: "mongodb://mongo:27017/conduit"

authentication:
  enabled: true

router:
  enabled: true
  ingress:
    enabled: true
    host: api.yourdomain.com

adminPanel:
  enabled: true
  ingress:
    enabled: true
    host: admin.yourdomain.com
```

### 3. Deploy

```bash
helm install conduit conduit-platform/conduit -f values.yaml
```

### 4. Verify Deployment

```bash
kubectl get pods
kubectl get services
kubectl get ingress
```

## Configuration Options

### Core Settings

```yaml
core:
  replicas: 2
  resources:
    requests:
      memory: "256Mi"
      cpu: "250m"
    limits:
      memory: "512Mi"
      cpu: "500m"
  env:
    MASTER_KEY: "your-secure-master-key"
    GRPC_KEY: "your-grpc-secret"
```

### Module Configuration

Enable/disable modules as needed:

```yaml
authentication:
  enabled: true
  replicas: 2

database:
  enabled: true
  
chat:
  enabled: true

email:
  enabled: false

storage:
  enabled: true
```

### Ingress Configuration

```yaml
router:
  ingress:
    enabled: true
    host: api.yourdomain.com
    annotations:
      kubernetes.io/ingress.class: nginx
      cert-manager.io/cluster-issuer: letsencrypt-prod
    tls:
      enabled: true
      secretName: api-tls
```

## Local Development (Minikube)

### 1. Start Minikube

```bash
minikube start
minikube addons enable ingress
```

### 2. Get Minikube IP

```bash
minikube ip
```

### 3. Update /etc/hosts

Add these entries (replace IP with minikube IP):

```
192.168.49.2  api.conduit.local
192.168.49.2  admin.conduit.local
```

### 4. Deploy with Local Values

```yaml
# values-local.yaml
router:
  ingress:
    host: api.conduit.local

adminPanel:
  ingress:
    host: admin.conduit.local
```

```bash
helm install conduit conduit-platform/conduit -f values-local.yaml
```

## Production Checklist

### Security

- [ ] Set strong `MASTER_KEY`
- [ ] Enable `GRPC_KEY` for internal communication
- [ ] Use TLS for all ingress
- [ ] Configure network policies
- [ ] Set resource limits

### High Availability

- [ ] Run multiple replicas of each service
- [ ] Use external Redis cluster
- [ ] Use managed database service
- [ ] Configure health checks
- [ ] Set up pod disruption budgets

### Monitoring

- [ ] Enable metrics endpoints
- [ ] Configure Prometheus scraping
- [ ] Set up Grafana dashboards
- [ ] Configure alerting

### Backup

- [ ] Database backup strategy
- [ ] Redis persistence (if needed)
- [ ] Configuration backup

## Upgrading

### Update Values

```bash
helm upgrade conduit conduit-platform/conduit -f values.yaml
```

### Check Status

```bash
kubectl rollout status deployment/conduit-core
```

## Troubleshooting

### View Logs

```bash
kubectl logs -l app=conduit-core
kubectl logs -l app=conduit-authentication
```

### Check Events

```bash
kubectl get events --sort-by='.lastTimestamp'
```

### Access Pod Shell

```bash
kubectl exec -it POD_NAME -- /bin/sh
```
