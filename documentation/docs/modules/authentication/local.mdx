#  Local

First of all, you have to create a client , to proceed to an operation.
See here for more: [Create a client](../../core/security)

**Every request should contain**:

| Param      | Required |       Type |
| ----------- | ----------- | ----------- |
| masterkey      | True       |  Header
| clientsecret   | True        |  Header
| clientid       | True        |  Header
| Authorization   | True        |  Header
**When a register and login occurs the authorization header shouldn't be provided**
## User registration
**Request**
```tsx
curl --location --request POST '<<CONDUIT_URL>>/authentication/local/new' \
--header 'masterkey: <<MASTERKEY>>' \
--header 'clientid: <<CLIENT_ID>>' \
--header 'clientsecret: <<CLIENT_SECRET>>' \
--header 'Content-Type: application/json' \
--data-raw '{
    "email": "<<YOUR_EMAIL>>",
    "password": "<<YOUR_PASSWORD>>"
}'
```
**Response**
```json
{
    "userId": "AUTOGENERATED_USER_ID"
}
```
## Verify email
Since  a user is registered, you need to verify your email. ***Check your email to take the verification token***!

| Param      | Required |       Type |
| ----------- | ----------- | ----------- |
| verificationToken         | True  |   Url

```tsx
curl --location --request GET '<<CONDUIT_URL>>/hook/authentication/verify-email/' \
--header 'clientid: <API Key>'
```
## Login

| Param      | Required |       Type |
| ----------- | ----------- | ----------- |
| email      | True        |   Body
| password   | True        |  Body

**Request**
```tsx
curl --location --request POST 'http://localhost:3000/authentication/local' \
--header 'masterkey: <<MASTERKEY>>' \
--header 'clientsecret: <<CLIENT_SECRET>>' \
--header 'clientid: <<CLIENT_ID>>' \
--header 'Content-Type: application/json' \
--data-raw '{
    "email": "<<YOUR_EMAIL>>",
    "password": "<<YOUR_PASSWORD>>"
}'
```
**Response**
```json
{
    "userId": "USER_ID",
    "accessToken": "<<YOUR_ACCESS_TOKEN>>",
    "refreshToken": "<<YOUR_REFRESH_TOKEN"
}
```
