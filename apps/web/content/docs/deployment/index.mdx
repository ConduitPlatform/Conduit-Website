Choose how to run Conduit based on your environment and operational needs.

## Decision matrix

| Mode | Best for | Complexity |
|------|----------|------------|
| [Local from source](/docs/deployment/local-source) | Contributing to Conduit core or modules | High |
| [Docker Compose](/docs/deployment/docker-compose) | Local dev and small self-hosted stacks | Low |
| [Kubernetes / Helm](/docs/deployment/kubernetes-helm) | Production clusters, GitOps | Medium |
| [Container images](/docs/deployment/container-images) | CI/CD, pinning versions, multi-arch pulls | Reference |

## Prerequisites (all modes)

- **MongoDB** or **PostgreSQL** for the database module
- **Redis** for core bus and realtime
- Node.js **24+** only when building from source or running the CLI locally

## After deployment

1. Enable modules you need via Admin API or [MCP](/docs/getting-started/mcp-setup)
2. Provision schemas and config at dev time — not from app runtime
3. Point applications at the **Client API** (default port 3000)
